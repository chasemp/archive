RIPE Atlas Getting Started

Wednesday, April 08, 2015 » internet, ripe

Preamble

RIPE NCC is one of the Regional Internet Registries. There are 5 (RIR's) and RIPE NCC
serves Europe. RIPE NCC runs one of the neatest open projects on the Internet today called
RIPE ATLAS.

“With your help, the RIPE NCC is building the largest
Internet measurement network ever made. RIPE Atlas
employs a global network of probes that measure Internet
connectivity and reachability, providing an unprecedented
understanding of the state of the Internet in real time.”

Imagine we have 8000+ probes scattered around that all
report metrics back to a central place. The data would
allow us to get a holistic view of overall trends, outages,
and performance. This is precisely RIPE Atlas.

There are a few ways to get involved in RIPE Atlas:

  • Host a probe (A small hardware device that collects metrics)
  • Host an anchor (A larger device
  • Sponsor the effort
  • Become an Ambassador

As you host a probe, an anchor, or sponsor you get //credits//. Credits are a system to allow
those who contribute to benefit. Each User-Defined Measurement will cost Credits.

Understanding User-Defined Measurements and Credits

Atlas allows users of the system to earn Credits by participating. These credits are in turn
used to purchase custom actions. If you want to see the result of 10 probes world wide checking
the SSL certificate of your endpoint it will cost 200 Credits. There are a few controls to keep
any one user from abusing the system: max number of parallel measurements (100),
maximum number of probes per measurement (500), and maximum number of measurements
towards the same target (10).

If we create a measurement in the UI the system will
indicate how many points we have currently and how long the measurement can run at the current interval.

Examples of use by the day for a measurement every 10 minutes from 50 global probes:

An IPv4 ICMP ping will use 21,600 credits
An IPv4 ICMP Traceroute will use 144,000 credits

Results from a UDM can be specified as public (default) or private. I believe probe existence
in general can be hidden if it is undesirable to have random users running measurements.

Anchors

My current employer hosts 3 anchors:

  • https://atlas.ripe.net/probes/6092/
  • https://atlas.ripe.net/probes/6093/
  • https://atlas.ripe.net/probes/6095/

Anchors come with some constant measurements that do not cost in credits (one of
the nice advantages of hosting an anchor). A seismograph for the ICMPv4 Ping
for 6092 can be seen here: https://atlas.ripe.net/probes/6095/. There is also
a constant ICMPv6 Ping, v4 Traceroute, and v6 Traceroute. Anchors also earn
Credits at a rate higher than probes.

Running a UDM

If you are hosting a probe now and earning credits you can login and generate
an API key for using the REST interface.

https://atlas.ripe.net/keys/

Launch a one time TCP traceroute from 5 random world wide hosts to en.wikipedia.org:


curl --dump-header - -H "Content-Type: application/json" -H "Accept: application/json" -X POST -d {
 "definitions": [
  {
   "target": "en.wikipedia.org",
   "af": 4,
   "timeout": 4000,
   "interval": 900,
   "description": "Traceroute measurement to en.wikipedia.org",
   "protocol": "TCP",
   "resolve_on_probe": false,
   "packets": 3,
   "size": 48,
   "firsthop": 1,
   "destination_option_size": 0,
   "port": 443,
   "paris": 16,
   "maxhops": 32,
   "hop_by_hop_option_size": 0,
   "dontfrag": false,
   "type": "traceroute"
  }
 ],
 "probes": [
  {
   "tags": {
    "include": [],
    "exclude": []
   },
   "type": "area",
   "value": "WW",
   "requested": 5
  }
 ],
 "is_oneoff": true
} https://atlas.ripe.net/api/v1/measurement/?key=YOUR_KEY_HERE

The results of this test can be seen in the UI or
fetched from the CLI:

curl https://atlas.ripe.net/api/v1/measurement/1945703/result/
