linux permissions

Tuesday, March 24, 2015 » linux, permissions, sudo

This Post is A Work in Progress

There are a ton of linux permissions references around but I have referenced a new
nuances lately and it has prompted me to write my own :) Both as a way to dig into
a fun topic and as somewhere to point people with questions I trust. Best beginning
overview https://en.wikipedia.org/wiki/File_system_permissions but I have noticed
that what happens is a user comes across an aspect of permissions they have never seen.

Suddenly they question everything they know, or maybe that's just me.

Tools:

chmod
chown

Chmod

We want to change what abilities for the existing categories of:

  • owner
  • group
  • everyone

The same settings have different meanings in the case of files and folders.

Chown

We want to change who the owner and/or group refers too.

visudo

We want to change what escalated permissions a user can access safely.

root ALL=(ALL) ALL

su

Becoming or executing commands as another user

chattr (i.e. can root protect a file so well even they can't remove it?)

Change attributes on a file (usually to set an immutable bit)

chattr +i filename.ext or remove chattr -i filename.ext

setcap

Setting capabilities

setuid

Setting a uid for a particular file

setgid

Setting the gid for a particular file

concepts

CAP_DAC_OVERRIDE – capability root has to “ignore” permissions
