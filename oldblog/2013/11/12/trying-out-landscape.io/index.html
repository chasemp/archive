trying out landscape.io

Tuesday, November 12, 2013 » landscape, python

Preamble

I recently came across a service in beta scoring Python repositories on github.

Landscape

In their own words:

Landscape is a code quality inspector for Python codebases.
It inspects your code to find errors, possible problems, security issues.

Basically, you associate their service with your repo and they use Pylint and other tools to assign it an overall score based on PEP8 compatibility and other criteria.

Using Sup as an example

I have this very small project that a few people use in place of Ping so I pointed it towards that.

I received an email with my first report card:

As shown on the site:

54% is listed as an 'OK' rating.

What is it they are finding?

Most of what they found is cruft that should be cleaned up or removed.

The socket module being reimported in my code for different classes…

An unused import from a previous refactoring…

An unused exception assignment from debugging…

Improving my score

I addressed some of the concerns as seen here and got my new score.

Thoughts

Seems pretty nice, and it's free so I can't complain. Right now it's only applicable to public Python repositories on github though. The only thing I wasn't fond of was that after I had pushed my commit there was nothing on the site in my email saying they were running a new test. I thought for sure from the delay there was a problem.
