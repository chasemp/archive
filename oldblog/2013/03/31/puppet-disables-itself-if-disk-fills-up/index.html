puppet disables itself if disk fills up

Sunday, March 31, 2013 » puppet

Preamble

A quick note if someone is wondering why puppet has disabled itself. Make sure
your disks have not filled up. Not sure if only the /var partition will cause this
but in my case:

This machine is behind on it's configuration.

df -Th

Filesystem    Type    Size  Used Avail Use% Mounted on
/dev/sda1      xfs     94G  91G   91G   100% /

puppet agent --test

Skipping run of Puppet configuration client; administratively disabled

Once the space has been cleared

puppet agent --enable && puppet agent --test
