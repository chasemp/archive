ssh go forth and do my will

Tuesday, August 21, 2012 » bash, ssh

Preamble

Commands run via SSH silently as a background process.

Occassionally, I need to run a command on a remote host and I don't have time to wait for the output. If success or
failure isn't a big deal I sometimes use the following method.

ls remotely

Let's say I badly need a recursive ls on a box

ssh <me>@<server> 'nohup ls -R &> /tmp/ls.txt &'

Breaking it down

ssh         -- obviously you need the client and server
<me>@<server>   -- Please be using key based authentication
''          -- ssh will accept a one and done command instead of going fully interactive
nohup       -- tell the command do not listen to HUP signal
ls -R       -- for fun we need that file list
&>          -- redirect output.  this is a newer version of this.
/tmp/ls.txt     -- for me to look at later.  note /tmp does not survive reboots.
&           -- run me in the background!

Looks like

rush:~$ ssh rush@<server> 'nohup sudo ls -R &> /tmp/ls.txt &'

No output to speak of…like I said just go do it.

What happened

rush:~$ ssh rush@<server> 'cat /tmp/ls.txt'
.:
1.TXT
bin
git
test.txt
