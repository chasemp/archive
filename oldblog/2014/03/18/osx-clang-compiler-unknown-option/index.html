osx clang compiler unknown option

Tuesday, March 18, 2014 » clang, mynt, osx

As of OSX Xcode tools <=5.1 the clang compiler treats any unknown flag as an error.
This can cause some issues with existing tools that have old flags, or uncommon flags,
or who knows what.

You see something like:

clang: error: unknown argument: '-mno-fused-madd' [-Wunused-command-line-argument-hard-error-in-future]

I see this when doing:

#on maverick
pip install mynt

It is an easy fix for now to ignore this for an oneoff:

ARCHFLAGS=-Wno-error=unused-command-line-argument-hard-error-in-future pip install mynt

I have read you can also ditch the OSX native python and install it from brew and things work out.
